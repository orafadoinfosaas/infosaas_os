
**3 perfis (cards):**

**Produto lançado, equipe desmobilizada**
O projeto foi entregue, o time foi liberado. Agora surgem ajustes, melhorias e bugs — sem ninguém para responder com agilidade e conhecimento do projeto.

**Evolução constante no roadmap**

**Operação que não pode parar**
O sistema é crítico para o negócio. Uma falha não resolvida rapidamente tem custo real — financeiro e de reputação. Prioridade de atendimento não é negociável.

**Ajustes de design sob demanda**
Melhorias de interface, identidade visual ou UX dentro da franquia mensal — sem contratar um designer separado.

**Evolução de código**
Novas funcionalidades, integrações, otimizações de performance ou refatorações dentro do volume mensal contratado.

**Registro detalhado de atividades**
Documentação de tudo que foi alterado, com contexto. O produto não fica na cabeça de uma pessoa — fica registrado.

## Seção 4 — Por que a continuidade importa

**Eyebrow:** `POR QUE ISSO IMPORTA`

**Headline:**
```
Produto parado
é produto que regride.
```

**Corpo:**
O mercado muda, a base de usuários cresce, surgem integrações novas. Sem evolução contínua, o que foi construído com cuidado começa a ficar para trás — e o custo de recontratar do zero, explicar o projeto e reconstruir contexto é muito maior do que manter.

**2 pontos de atenção:**

**O risco da pessoa-chave**
Se existe alguém que sabe como tudo funciona — e só essa pessoa — qualquer ausência vira crise. O Suporte Premium distribui esse conhecimento para um time com registro auditável.

**Custo de recontratar vs. custo de manter**
Recontratar, repassar contexto e realinhar expectativas tem custo financeiro e de tempo. A franquia mensal mantém o time aquecido e o projeto evoluindo com previsibilidade.
