# Seção 06 — Fotografia e Imagem

> **Propósito desta seção no brandbook:** Documentar o estilo visual oficial para fotografia de pessoas, uso do símbolo como motivo decorativo e uso de screenshots/mockups de projetos. A Infosaas® usa dois tipos de imagem: fotografia com pessoas (em contexto específico) e capturas de interface (para mostrar o trabalho concreto).

---

## 06.1 Fotografia com pessoas

O estilo fotográfico da Infosaas® é definido por:

**Contexto:** Retratos individuais ou em duo — nunca grupos grandes  
**Fundo:** Papel pintado de laranja (`#FF3D00`) — não gradiente, não digital  
**Iluminação:** Natural ou simulada natural — luz quente lateral, sem flash duro  
**Tratamento de cor:** Contraste levemente aumentado, temperatura levemente aquecida  
**Enquadramento:** Meio-corpo a rosto — nunca fotos de longe, impessoais  
**Vestuário:** Casual profissional — sem terno/gravata, sem moletom de academia  
**Expressão:** Natural, receptivo, mãos visíveis quando possível  

### O que nunca usar em fotografia

| Proibido | Por quê |
|---|---|
| Fotos de banco de imagem com fundo branco estéril | Parecem genéricas — incompatível com premium |
| Fotos com filtro azul/frio | Incompatível com a paleta quente da marca |
| Fotos monocromáticas (P&B) | Fora do sistema de marca |
| Fotos com muitas pessoas (grupos > 3) | Perdem o foco e a identidade individual |
| Ilustrações de personagens abstratos | Incompatível com o posicionamento direto e humano |
| Mockup 3D genérico de device | Usar screenshots reais no contexto correto |

## 06.2 O símbolo como motivo decorativo

Quando não há foto de pessoa, o símbolo da Infosaas® é usado como elemento visual de preenchimento e textura.

**Como usar:**

| Parâmetro | Valor |
|---|---|
| Tamanho | 60–120% da altura do frame |
| Posição | Offset — parcialmente fora do frame, nunca centralizado e completo |
| Cor | `#B82C00` (`--laranja-deep`) sobre fundo laranja — overlay de baixo contraste |
| Opacidade | O efeito é de baixo contraste intencional — não precisa de opacity CSS, é de cor |
| Superfície | Exclusivamente sobre fundo laranja `#FF3D00` |

**O que não fazer com o símbolo decorativo:**
- Não usar símbolo decorativo sobre fundo branco ou preto (perde a sutileza do overlay)
- Não centralizar e completar o símbolo — deve ser cortado pelo frame
- Não usar o símbolo decorativo sem o fundo laranja
- Não combinar símbolo decorativo com foto de pessoa no mesmo frame

## 06.3 Screenshots e mockups de interface

Quando o objetivo é mostrar um projeto, entrega ou produto concreto, usar captura real de interface — não mockup genérico.

**Diretrizes:**

| Parâmetro | Diretriz |
|---|---|
| Fonte | Screenshot real — do projeto entregue ao cliente |
| Device | Browser em desktop (1440px), ou frame de device neutro (sem marca de fabricante) |
| Fundo do frame | Branco-off `#F5F5F5` — nunca gradiente colorido ao redor do device |
| Tratamento | Sem filtro, sem efeito. Limpeza de UI se necessário (remover dados sensíveis) |
| Contexto | Sempre com título/legenda que contextualiza o projeto |
| Quantidade | 1–3 screenshots por projeto — não galeria extensa |

**Tipos de screenshot aprovados:**
- Dashboard ou interface de produto digital
- Tela de landing page construída
- Fluxo de onboarding ou formulário
- Relatório ou painel de dados

**O que não usar:**
- Mockup 3D de iPhone/Macbook com reflexo ou sombra dramática
- Tela com dados reais de cliente sem autorização
- UI incompleta ou com placeholder visível ("Lorem Ipsum")
- Tela com bugs ou estados de erro (a menos que seja contexto educacional)

## 06.4 Hierarquia de uso por superfície

| Superfície | Imagem recomendada |
|---|---|
| Hero de campanha | Foto com pessoa sobre laranja |
| Seção "Sobre" ou "Time" | Foto com pessoa sobre laranja |
| Case de projeto | Screenshot de interface + título |
| Banner de redes sociais (sem projeto) | Símbolo decorativo sobre laranja |
| Apresentação comercial — capa | Foto com pessoa ou símbolo decorativo |
| Apresentação comercial — slide de projeto | Screenshot de interface |
| Documento/contrato | Sem imagem — apenas logo e tipografia |
